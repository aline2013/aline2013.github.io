梅沙教育App体验报告

发表于 2018-07-10 | 分类于 产品 |

一、体验环境

1.产品名称： 梅沙教育App
2.版本号：v2.1.01
3.体验机型：iPhone 6s, iOS 11.4
4.网络环境：WiFi & 4G
5.体验身份：一名没有学员的登录者
6.体验时间：2018年7月6日~2018年7月10日

二、产品简介

1.产品简介

梅沙教育App是一款具有社区属性的、专注于营地教育的移动电商类产品。家长可以根据孩子的年龄、兴趣爱好和空闲时间选择购买合适的营地教育课程，在加入课程后，可以在App上看到孩子的上课和参营学习的动态，同步感受到孩子的进步，同时可以对动态进行点赞、评论、收藏和转发，形成一个学习社区。

2.Slogan

走出去长大，玩乐中学习。

3.产品定位

梅沙教育是一个致力于打造一个安全且专业的体验式教育营地，为3 到18岁的青少年提供优质的教育服务。而梅沙教育App是梅沙教育用于是连接平台、家长和学员的中间工具，提倡“互联网+教育”的理念，移动端的梅沙教育App为家长购买课程和查看孩子动态提供了一个随时随地，更为便捷的途径。

三、目标用户及典型场景

1.目标用户

作为连接平台、家长和学员的梅沙教育App，目标用户自然是家长和孩子。
由于梅沙教育App是2016年发布的年轻App，没有能从数据网站上找到用户群体的相关参数，所以，我自行分析其用户目标。
根据梅沙教育的校区分布和课程价格， 得到以下猜测。

  1. 用户地域：一线城市和部分新一线、二线城市。
  2. 用户年龄： 为人父母的群体，年龄在25~45岁之间。青少年儿童群体，年龄在3~18岁之间。
  3. 消费能力：用户追求高品质教育，经济实力强，消费能力高。

2.典型场景

  1. 夫妻两人都是公司的高层，总有忙不完的工作，正值小孩放暑假，2人抽不出时间来陪伴小孩，但又不希望他没人陪伴，真好借此机会，报名梅沙教育App的夏令营，还可以增长孩子的见识。
  2. 小明在看《爸爸去哪儿》时，希望能像节目中的小朋友一样，和父母去不同的地方体验生活，于是跟父母说明了想法，于是父母通过梅沙教育App下单了亲子课程。
  3. 老王一直相信纸上得来终觉浅，希望小孩能通过实践学到更多实用的东西，于是每个月都在梅沙教育App给孩子报名一次周末营。

四、关键功能体验

在APP的页签栏有四个页面选项，它们分别是“动态”、“学员”、“发现”、“我的”。

本质上，关键功能是以下三点：
“动态”对应的是社区。
“发现”对应的是商品，即课程。
“学员”，“我的”对应的是服务。

1.社区

社区包含学员的课程和参营动态和梅沙社区营地官方内容推送。打开App就能感受到营地的气氛和孩子的喜怒哀乐。

2.商品

在“发现”页，户外营地课程推荐展示了4个快速入口：帆船、周末营、潜水和综合，原因猜测有2，筛选课程类型中用户最常选择的条件， 或平台认为这样可以突出户外营地教育的主题，这样的设置，容易激起了用户户外运动的兴奋感和刺激感。
城市征途是区分不同城市的活动，便于用户直接找自己所在城市的活动。
最热课程也为用户提供了便捷入口。

体验差的部分：
  1. 如以上所写，课程有多个便捷入口，如“OW开放水域潜水课程”，即属于三亚城市分类，又属于潜水分类，三亚城市必然容易多水上课程，导致，两个入口打开的课程列表中大部分重复，难以找到想要的课程，只能通过自行拆解标题，在标题中找到课程类型的关键字，如果有固定字段展示好些。
  2. 购买课程时，选择了学员信息，直接跳转到信息表单页，开始以为是确认信息，多试几次才能意识到学员信息不完整，需要完善。
  3. 营地生活回顾，指营地日记，点开是视频列表，不像是长期更新的内容，和“发现”的含义有冲突，作为课程总结，如果有文字可以加以描述，可减少用户流量消耗，提高用户查看率。
  4. 课程筛选，点多几次就提示：操作频繁。查看课程详情出现明显卡顿，部分视频有60s的广告，分享课程出现错误页面，影响体验。

3.服务

在“学员”页面中，为用户提供学员的课程通知等服务。精选专题和“发现”页重复。
在“我的”页面中，为用户提供添加学员和客服等服务。

五、总结

由于没有添加有效学员，所以没有办法获得到购买课程后的体验，所以忽略这部分。
目前来看，梅沙教育App不同于在线教育平台，它专注于线下和户外教育，采用营地教育的方式，对孩子寓教于乐，真正地让孩子们在快乐中学习成长。虽然，梅沙教育App提供了一套相对完整的购买课程和课程服务，但个人感觉功能细节和交互细节还欠缺考虑。希望梅沙教育能在完成功能的基础上，有自己的设计风格，这样更符合高端消费的style。

AMP（Accelerated Mobile Pages），加速移动网络网页

AMP是什么

通过实行一套合理的规则，优化你的网页主体内容。本质是代码减负+缓存预加载，严格控制网页装载内容大小和顺序，达到明显提升用户体验的技术。

示例 - AMP 官网

AMP的组成

AMP包括三个部分：

  1. AMP HTML： 新定义了一套HTML标签，如amp-img, amp-video。使用这些tag可以确保加载的顺滑。同时amp也负责管理资源何时加载，避免不必要的流量。
  2. AMP Runtime： 完全由js实现，实现了所有的AMP性能优化实践。其中最大的优化就是保证外部所有资源都是异步装载，另外还包括Sandbox处理iFrame的模块，资源装载的同时预先计算出每一个元素的位置和大小；禁用慢的CSS样式；
  3. Google AMP Cache： 提供第三方缓存，为了推动AMP，谷歌提供免费的全球CDN服务，可以缓存图片，静态文件等，使用HTTP2.0协议保证缓存的高效率。从Google的搜索结果里进入AMP网页就可以做到预加载甚至预渲染，从来带来极速体验。Google 聲稱 AMP 的頁面在搜索結果里排名會更靠前。

此外， AMP validator 是一个校验器。为了确保网页能够达到所要求的极速，AMP对网页里能使用的东西做了严格的限制。比如不允许使用自己的JS，不允许外部加载CSS。校验器是为了帮助你检查你的网页是否达到了这些要求。

PWA

发表于 2018-06-01 | 分类于 技术 |

PWA是什么

Progressive Web App(渐进式网页应用)，简称 PWA，是由Google提出的，提升Web App的使用体验的一种新方法。

理解PWA
PWA不是特指某一项技术，而是应用了多项前沿技术的 Web App。其核心技术包括 App Manifest、Service Worker、Web Push等等。

渐进式：核心技术的浏览器支持度不高，如果你构建一个 PWA，即使在一个不支持的老旧浏览器上运行，它仍然可以作为一个普通的网站来运行。只有在支持这些新功能的浏览器中才会增强体验。

示例 豆瓣网页版（安卓手机）

PWA 核心技术

  1. Web App Manifest， 即通过一个名为manifest.json清单文件向浏览器暴露 web app的元数据，包括名字、icon 的 URL 等，以备浏览器使用。比如在添加至主屏或推送通知时暴露给操作系统，从而增强 web 应用与操作系统的集成能力。
  2. Service Workers ，是一个可编程的 Web Worker，实际上是一段在后台运行的脚本，独立线程，独立于当前网页进程，有独立的 worker context。
    Service Worker 的出现是为了实现web app的离线使用、消息推送的功能。
    它就像一个位于浏览器与网络之间的客户端代理(空中管制员)，可以拦截和处理流经的http请求，可以缓存文件，缓存的文件可以被网页进程取到（包括网络离线状态）。
  3. Push Notification，Push API 的出现则让推送服务具备了向 web 应用推送消息的能力。

数据分析网站相关参数（PV,UV...）

发表于 2018-05-02 | 分类于 产品 |

参数

PV（page view，页面浏览量）
用户每打开1个网站页面，记录1个PV。用户多次打开同一页面，PV值累计多次。主要用来衡量网站用户访问的网页数量。是评价网站流量最常用的指标之一。

UV（ unique visitor，网站独立访客）
通过互联网访问、流量网站的自然人。1天内相同访客多次访问网站，只计算为1个独立访客。该概念的引入，是从用户个体的角度对访问数据进行划分。

IP（独立IP）
1天之内（00:00~24:00），访问网站的不重复的IP数。拥有特定唯一IP的计算机访问网站的次数，因为这种统计方式比较容易实现，具有较高的真实性。

VV（video view，视频播放量）
一个统计周期内，视频被打开的次数。

CV（Content Views，内容播放数）
一个统计周期内，视频被打开，且视频正片内容（除广告）被成功播放的次数。

VV和CV属于播放类指标，PV、UV、IP属于浏览类指标。

Traffic 流量
Engagement （参与度）
Esitmated visits 估计访问量
time on site 停留时间
Bounce rate （跳出率）
只浏览一个网页的访问量占总访问量的比率

简称

SEO （Search Engine Optimization）
搜索引擎优化

SEM（Search Engine Marketing）
搜索引擎营销,竞价排名

ASO (App Store Optimization) ，参数如ASO关键字
应用商店搜索优化, 移动APP的SEO优化，应用商店的搜索结果优化排名

ASM（App Store Marketing）
应用商店搜索付费营销
